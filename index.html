🗺️ 학습 01

상태 공간과 탐색

문제 해결을 위해 인공지능은 어떻게 탐색할까요? 핵심 개념을 먼저 이해해 봅시다.

🚩 초기 상태 (Initial State)

문제가 주어진 시작 상태입니다. 탐색의 출발점이 되는 상태로, 문제의 처음 조건이 주어진 상태를 의미합니다.

⚙️ 상태 정보 (Action)

어떤 상태에서 다른 상태로 변경되는 데 적용되는 정보나 비용입니다. 상태 전환에 필요한 연산이나 이동을 의미합니다.

🌐 상태 공간 (State Space)

초기 상태와 목표 상태를 포함하여 문제 해결 과정에서 나타나는 모든 상태의 집합입니다.

🏁 목표 상태 (Goal State)

문제를 해결한 마지막 상태입니다. 탐색이 성공적으로 끝났을 때 도달하는 최종 상태를 의미합니다.

🔍 탐색 (Search)

초기 상태에서 출발하여 상태 정보를 기초로 다양한 상태를 조사하고, 목표 상태에 이르는 경로를 찾아다니며 문제의 해답을 구하는 것입니다. 인공지능 문제 해결의 핵심 원리입니다.

🗼 문제 적용: 하노이의 탑 (Tower of Hanoi)

정의: 세 개의 막대 중 한 막대에만 원반들이 크기 순으로 쌓여 있는 상황에서, 이 원반 모두를 다른 두 막대 중 어느 하나의 막대로 옮기는 문제입니다.
적용 가능한 연산: move(p1, p2), move(p1, p3), move(p2, p1), move(p2, p3), move(p3, p1), move(p3, p2)

  • 1원반은 한 번에 하나씩만 옮길 수 있다.
  • 2큰 원반은 작은 원반 위에 올릴 수 없다.
  • 3원반을 옮기는 횟수가 최소가 되도록 한다.

🌲 학습 02

탐색 공간의 구조화

탐색 공간을 트리 구조로 나타내면 문제 해결 경로를 체계적으로 파악할 수 있습니다.

🌳 트리 (Tree)

뿌리와 가지, 잎을 가진 나무를 거꾸로 세운 형태의 정보 구조입니다.

⬤ 노드 (Node)

트리를 구성하는 각각의 데이터로, 보통 점이나 원으로 표현합니다.

👑 루트 노드 (Root)

부모가 없는 최상위 노드로, 탐색에서는 초기 상태를 의미합니다.

➖ 간선 (Edge)

노드와 노드를 연결하는 선으로, 탐색에서는 상태 간의 관계(전환)를 의미합니다.

⬆️ 부모 노드 (Parent)

두 노드가 간선으로 직접 연결된 상위의 노드입니다.

⬇️ 자식 노드 (Child)

두 노드가 간선으로 직접 연결된 하위의 노드입니다.

📏 깊이/레벨 (Depth)

자신을 제외한 조상 노드의 개수입니다. 루트 노드의 깊이는 0입니다.

💡 관계의 상대성

노드 간의 관계는 상대적입니다. 부모이면서 동시에 자식일 수 있고, 자식이 없는 노드(리프 노드)도 있습니다.

🖱️ 트리 예시 — 노드를 클릭해 보세요!

각 노드를 클릭하면 해당 노드가 루트인지, 부모와 자식 노드가 무엇인지 확인할 수 있습니다. 오른쪽의 숫자는 깊이(레벨)입니다.

🗼 하노이의 탑 상태 공간 트리 (원반 2개)

▶ 버튼을 클릭하면 원반 2개일 때 하노이의 탑 상태 공간 트리가 단계별로 만들어집니다. 점선 테두리 노드는 이미 나온 상태(반복 상태)를 나타냅니다.
