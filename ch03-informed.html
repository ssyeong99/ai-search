정보 이용 탐색

목표까지의 거리 정보(휴리스틱)를 활용하여 더 똑똑하게 탐색하는 방법을 알아봅시다.

🎮 탐색이 적용되는 게임 체험하기 🧩 8퍼즐 🗼 하노이의 탑 ⭕ 틱택토
🎯 학습 목표
  • 1정보 이용 탐색의 핵심 개념인 휴리스틱을 설명할 수 있다.
  • 2최상 우선 탐색의 원리를 이해하고 탐색 과정을 따라갈 수 있다.
  • 3A* 알고리즘의 f(n) = g(n) + h(n) 공식을 이해하고 맹목적 탐색과 비교할 수 있다.
🧠 학습 01

정보 이용 탐색이란?

목표에 대한 사전 정보(휴리스틱)를 활용해 불필요한 탐색을 줄이는 방법입니다.

💡 휴리스틱(Heuristic)이란? 목표 상태까지의 예상 비용이나 거리를 추정한 값입니다. 정확한 값은 아니지만, 탐색 방향을 결정하는 데 도움을 줍니다. 예) 지도에서 직선 거리로 목적지까지의 거리 추정.
Best-First Search

최상 우선 탐색
Greedy Best-First Search

열린 노드 중 휴리스틱 값 h(n)이 가장 작은 — 즉, 목표에 가장 가까워 보이는 — 노드를 먼저 탐색합니다. 빠르지만 최적 경로를 보장하지 않습니다.

h(n) 🎯 목표 지향 빠르나 비최적
A* Algorithm

A* 알고리즘
A-Star Algorithm

출발점에서의 실제 비용 g(n)과 목표까지의 예상 비용 h(n)을 합산한 f(n) = g(n) + h(n)이 가장 작은 노드를 선택합니다. 최적 경로를 보장합니다.

f(n)=g(n)+h(n) ⚡ 최적 보장 우선순위 큐

🌿 학습 01-③

최상 우선 탐색

평가 함수로 각 상태의 가치를 평가하여 목표에 가장 가까운 상태를 우선 탐색합니다.

최상 우선 탐색 : 현재 상태에서 목표 상태까지 가장 가까운 상태를 탐색하는 방법
평가함수를 사용하여 각 상태의 가치를 평가하고, 목표 상태에 가장 가까운 상태를 우선적으로 탐색한다.
★ 평가 함수(휴리스틱 함수) : 각 상태가 목표에 얼마나 가까운지를 예측하는 함수
f(n) = 목표와 다른 타일 수
평가 함수 f(n) = 목표 상태의 숫자 위치와 일치하지 않는 타일의 개수
f(n)이 작을수록 목표에 가까운 상태 → 최상 우선 탐색은 f(n)이 가장 작은 노드를 먼저 탐색

🗺️ 학습 02

A* 알고리즘을 이용하여 최단 경로 찾기

집에서 학교까지 가는 최단 경로를 A* 알고리즘으로 직접 찾아봅시다.

📍 다음은 집 근처에 있는 여러 장소 사이의 거리를 나타낸 약도입니다. 집에서 학교까지 가는 최단 경로를 A* 알고리즘으로 찾아보세요.
❶ 실제 거리
❶ 추정 거리 h(n) — 학교까지 직선 추정
📐 A* 알고리즘의 평가 함수
f(n) = g(n) + h(n)
g(n) : 시작 노드에서 노드 n까지의 실제 거리
h(n) : 노드 n에서 목표 노드까지의 추정 거리
1단계 집에서 이동 가능한 장소의 f(n) 계산
집(출발)에서 갈 수 있는 식당, 공원, 학원의 f(n) = g(n) + h(n)을 계산해보세요.
2단계 식당 선택 → 식당에서 이동 가능한 장소의 f(n) 계산
f(n) 최솟값인 식당(f=15)을 선택! 식당에서 갈 수 있는 슈퍼마켓, 공원의 f(n)을 계산해보세요.
3단계 공원 선택 → 공원에서 이동 가능한 장소의 f(n) 계산
f(n) 최솟값인 공원(f=15)을 선택! 공원에서 갈 수 있는 슈퍼마켓, 학원의 f(n)을 계산해보세요.
4단계 슈퍼마켓 선택 → 학교까지의 f(n) 계산
f(n) 최솟값인 슈퍼마켓(f=15)을 선택! 슈퍼마켓에서 학교까지의 f(n)을 계산해보세요.
🏆 최단 경로 발견!
A* 알고리즘으로 찾은 집에서 학교까지의 최단 경로입니다.
🏠 집 → 🍽️ 식당 → 🌳 공원 → 🛒 슈퍼마켓 → 🏫 학교
총 실제 거리: 집→식당(3) + 식당→공원(6) + 공원→슈퍼마켓(4) + 슈퍼마켓→학교(2) = 15

탐색 방법 비교

맹목적 탐색과 정보 이용 탐색의 특징을 한눈에 비교해 봅시다.

구분 맹목적 탐색 (DFS/BFS) 최상 우선 탐색 A* 알고리즘
사용 정보 없음 h(n) — 목표까지 예상 비용 g(n) + h(n)
탐색 기준 순서/레벨 h(n) 최소 또는 최대 f(n) 최소 또는 최대
최적 경로 BFS만 보장 ❌ 보장 안 됨 ✅ 보장

🔍 맹목적 탐색

정해진 순서에 따라 모든 상태를 탐색하므로 알고리즘이 단순하여 비교적 간단한 문제는 정답을 쉽게 찾을 수 있습니다.
하지만 목표 상태에 도달하기까지 모든 경우를 살펴보기 때문에 많은 시간이 소요됩니다.

⭐ A* 알고리즘

휴리스틱 정보를 활용하여 불필요한 탐색을 줄이고, 최적의 경로를 통하여 목표 상태에 도달할 수 있습니다.
