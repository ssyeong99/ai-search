맹목적 탐색 vs 정보 이용 탐색

두 가지 탐색 방법의 차이를 비교하여 지능적 탐색의 원리를 이해해봅시다.

📊 두 탐색 방법 비교
비교 항목 🔍 맹목적 탐색 (무정보 탐색) 🧩 정보 이용 탐색 (경험적 탐색)
사전 지식 활용 ❌ 사용하지 않음 ✅ 사전 지식 적극 활용
탐색 방식 모든 상태를 규칙에 따라 순서대로 탐색 다음 상태의 우선 순위를 결정하여 탐색
효율성 낮음 (불필요한 탐색 많음) 높음 (최적 경로 우선 탐색)
대표 알고리즘 너비 우선 탐색(BFS), 깊이 우선 탐색(DFS) A* 알고리즘, 탐욕적 최선 우선 탐색
적합한 상황 탐색 공간이 작고 정보가 없을 때 탐색 공간이 크고 사전 지식이 있을 때
Informed Search · 경험적 탐색

정보 이용 탐색

문제 해결 과정에서 경험이나 지식을 활용하여 탐색 효율을 높이는 탐색 방법.
휴리스틱 탐색 또는 경험적 탐색이라고도 한다.
* 휴리스틱(Heuristic) : 시간이나 정보가 불충분하여 합리적인 판단을 할 수 없거나 굳이 체계적인 판단이 필요 없는 상황에서 빠르게 사용할 수 있는 어림 짐작의 방법
정보 이용 탐색은 문제의 특성에 맞게 설계된 평가 함수인 휴리스틱을 사용하여 각 상태를 평가하고, 이 평가를 기반으로 탐색 방향을 결정한다.
* 평가 함수(휴리스틱 함수) : 각 상태가 목표에 얼마나 가까운지를 예측하는 함수. 상태의 가치를 추정하는 역할.

1 거리 기반 휴리스틱
▼
집까지의 직선 거리를 이용하거나 예상 소요 시간을 이용해서 더 빠르고 효율적으로 길을 찾는 방법. 항상 최적의 경로를 보장하지는 않아 때로는 예상 밖의 상황이 발생할 수 있다.
1현재 위치에서 가능한 모든 경로를 확인
2각 경로의 목적지까지의 직선 거리를 계산
3가장 짧은 직선 거리를 가진 경로 선택

체스에서의 수 싸움

체스 AI는 현재 말의 배치를 평가 함수로 점수를 매겨, 점수가 높은 수(상대에게 불리한 수)를 우선적으로 탐색합니다. IBM 딥블루는 이 방법으로 세계 챔피언을 이겼습니다.

⚫
바둑 대결

바둑은 경우의 수가 체스보다 훨씬 많아 맹목적 탐색이 불가능합니다. 구글 알파고는 휴리스틱 + 딥러닝으로 각 수의 가치를 예측하고, 이세돌 9단을 꺾었습니다.

🗺️
내비게이션 경로 탐색

구글 맵, 카카오맵 등은 목적지까지의 직선 거리 + 교통 상황을 휴리스틱으로 활용해 A* 알고리즘으로 최적 경로를 빠르게 찾아줍니다.

🤖
게임 AI

스타크래프트, 롤 등의 게임 AI는 현재 상황을 평가하여 가장 유리한 행동을 선택합니다. 단순히 모든 경우를 탐색하지 않고 지식 기반으로 빠르게 판단합니다.
